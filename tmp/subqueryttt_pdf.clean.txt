        What is a subquery?
 SQL allows for subqueries inside arbitrary 
   expressions 
 From a user perspective this is very powerful 
 e.g. suppose we want to find the student with the 
   highest score, we can use a subquery: 
  SELECT name
  FROM students
  WHERE score=
      (SELECT MAX(score) FROM students);
 This is an an uncorrelated subquery 
            What is a subquery?
 We can also use correlated subqueries 
 In this case, columns from the outer tables are used inside 
    the subquery 
 To find the highest scoring students per major, we can use the 
    following subquery: 
   SELECT name, major            SELECT name, major
   FROM students s1              FROM students s1
   WHERE score=                  WHERE score=
       (SELECT MAX(score)            subquery(s1.major);
        FROM students
        WHERE s1.major=major);
 This is a correlated subquery (s1.major is used from 
    inside the subquery) 
             What is a subquery?
 Logically, subqueries are executed for every row 
 Let
s resolve the previous subqueries with the

         Rule-Based Rewriting
 Many systems instead use heuristic rewrite rules to 
   flatten subqueries 
 Our query can be rewritten into the following query:
   SELECT s1.name,
          s1.major
   FROM students s1,
     (SELECT major, MAX(score) AS max_score
      FROM students
      GROUP BY major) s2
   WHERE s1.major=s2.major
     AND score=max_score;
 Not all subqueries are so easy to rewrite
       Heuristic Rewrite
 The result of these heuristic rewrite rules is that 
   queries that are rewritten execute fast 
 And queries that are not rewritten execute very slow 
 Execution time can jump from seconds to hours 
 Sometimes changing just a small part of the query 
   can remove ability to rewrite
